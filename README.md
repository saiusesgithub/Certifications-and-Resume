## Resume Versioning System

The resume is updated regularly, usually at the beginning or end of each month.

Each monthly folder contains a frozen snapshot of the resume at that point in time.

Example:

```text
Resume/2026/July/
Resume/2026/August/
Resume/2026/September/
```

This makes it possible to compare versions and track progress across:

- Work experience
- Internship roles
- Open-source contributions
- Technical projects
- Skills
- Leadership roles
- Achievements
- Certifications

---

## Latest Resume

The most recent application-ready resume is stored in:

```text
Resume/Latest/
```

This folder contains the version currently used for applications.

Typical contents:

```text
Sai_Srujan_Punati_Resume.pdf
main.tex
```

The PDF is the compiled resume, while `main.tex` contains the editable LaTeX source.

The contents of `Latest/` may be replaced whenever a newer resume becomes the primary application version. Older versions remain preserved in their respective year and month folders.

## Resume Source

The resume is written in **LaTeX** and compiled using **Overleaf**.

The template is based on a customized single-column software engineering resume format.

Each resume version normally contains:

```text
main.tex
resume.pdf
```

Additional files may be included when required:

```text
custom.sty
assets/
```

Source files are stored alongside the generated PDF so that every archived version remains reproducible and editable.

## Monthly Update Workflow

The following process is used for each monthly resume update:

1. Review new experience, projects, achievements, skills, and certifications.
2. Update the LaTeX source in Overleaf.
3. Compile and verify the final PDF.
4. Check links, dates, spelling, and formatting.
5. Create the relevant year and month folder.
6. Add the PDF and LaTeX source.
7. Update the `Latest/` folder with the current application-ready version.
8. Commit the update with a clear message.

Example commit message:

```text
docs: add July 2026 resume versions
```

For a targeted resume:

```text
docs: add July 2026 app development resume
```

---

## Versioning Principles

- Monthly folders are treated as permanent snapshots.
- Old resumes are not overwritten.
- `Latest/` always points to the current primary resume.
- Targeted versions are created only when they differ meaningfully.
- Each archived resume includes both the PDF and its LaTeX source.
- File and folder names use a consistent naming convention.
- Resume claims should remain accurate, verifiable, and current.
